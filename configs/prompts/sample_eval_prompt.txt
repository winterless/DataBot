## 评分参考（锚定尺度，请严格对照，可给出 0-10 间任意小数）

- **score=1**：纯乱码、token_ids、二进制/图像元数据，完全不可解析
- **score=2**：纯代码片段、法律条文、静态文档，无对话/推理结构，属噪音
- **score=3**：有基本结构但内容空洞，或单轮 instruction-only 无响应，Agent 无关
- **score=4**：单轮 QA、摘要、分类，可读但无多步推理或工具调用，Agent 相关性弱
- **score=5**：多轮对话但仅 system+user 无模型输出，或单轮 instruction+response 无交互
- **score=6**：含多轮对话+简单推理，或工具声明完整但无实际调用/执行痕迹
- **score=7**：多轮对话+推理链，有部分工具调用或环境反馈，格式基本规范
- **score=8**：完整 Thought-Action、多轮工具调用、环境交互，格式规范，高度适配 Agent
- **score=9**：在 8 基础上，含复杂异常处理、多步验证、分布式/具身等高级要素
- **score=10**：极少，近乎完美的 Agent 训练样本，覆盖全流程且质量极高

请将待评估样本与上述锚定对比后给出 score（可精确到 0.5，如 6.5），保持跨数据集尺度一致。

## 评估维度（综合考量）

1. 数据格式规范性（conversations、instruction/response 等）
2. 内容质量（完整、可读、有意义）
3. Agent 相关性（多步推理、工具调用、环境交互、代码执行等）
4. 可扩展性（样本是否代表该数据集典型形态）

## 输出格式（必须严格 JSON）

```json
{"score": <0-10 的数值，建议精确到 0.5>, "reason": "<一句话说明主要依据，不超过 50 字>"}
```
